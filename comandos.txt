git init: inicia um respositório local; 

git config: configura informações de usuário que estarão relacionadas aos commits

git init: inicia um repositório local e vazio referente ao diretório corrente;

git add: adiciona arquivos especificos ou todos os arquivos do diretório no staging da branch local. 

git status: retorna quais arquivos estão em staging(estado temporário de memória);

git rm: remove arquivos do staging.

git commit: salva os as versões dos arquivos em staging no repositório local.

git remote add: adiciona conexão do repositório local com um repositório remoto via url.

git push: envia atualizações do repositório local para o repositório remoto. 

git clone: constrói um repositório local que é clone de um repositório remoto, copiando todos os arquivos, branches e versões. 

git pull: atualiza a branch atual do repositório local com as modificações da mesma branch repositório remoto. 
    * git pull origin <nome da branch>: atualiza uma branch especificada, não necessariamente a atual.
